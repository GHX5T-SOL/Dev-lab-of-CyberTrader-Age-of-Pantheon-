## Task Summary

Implemented a stateful post-push regression monitor (`scripts/regression-monitor.mjs`) for the v6 repo. The monitor detects new commits on `origin/main` and automatically runs the regression suite (`typecheck + jest + health:live`). A launchd plist template enables periodic execution on the Mac mini.

### Deliverables

**New file ‚Äî `scripts/regression-monitor.mjs`**

Stateful Node.js monitoring script. Persists last-checked commit in `.git/regression-state.json` (not tracked by git).

Two modes:
- **Force mode** (`--force` / `npm run regression:check`): always runs the full suite
- **Monitor mode** (`npm run regression:monitor`): fetches `origin/main` and skips if no new commits since last check

Regression suite:

| Step | Command |
|---|---|
| typecheck | `npm run typecheck` |
| jest | `npm test -- --runInBand --forceExit` |
| health:live | `npm run health:live` |

Output: JSON to stdout. Exit 0 = pass or skip; exit 1 = regression or upstream error.

**New file ‚Äî `scripts/launchd/com.cybertrader.v6.regression-monitor.plist.template`**

LaunchAgent plist template for the Mac mini. Fires every 900 seconds (15 min). Instructions embedded:
1. Copy to `~/Library/LaunchAgents/` (drop `.template`).
2. Fill in `REPLACE_NODE_PATH`, `REPLACE_REPO_ROOT`, `REPLACE_LOG_DIR`.
3. `launchctl load ~/Library/LaunchAgents/com.cybertrader.v6.regression-monitor.plist`

**Modified ‚Äî `package.json`**

| Script | Command |
|---|---|
| `regression:check` | `node scripts/regression-monitor.mjs --force` |
| `regression:monitor` | `node scripts/regression-monitor.mjs` |
